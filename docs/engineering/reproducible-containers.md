# Reproducible Container Images

## Current State

Testcontainers in integration tests use default image versions, which may differ from production images. This can lead to subtle incompatibilities.

## Target State

Container images are declared once in `devenv.nix` and consumed by all components:

```
devenv.nix (single source of truth)
    │
    ├──→ environment variables
    │        │
    │        ├──→ testcontainers (MySQL::new(env!("MYSQL_VERSION")))
    │        │
    │        └──→ docker-compose (generated by nix)
    │
    └──→ sha256 hash verification
```

## Implementation Notes

1. **testcontainers**: Supports explicit image version via `MySQL::new("8.0.36")`
2. **Goal**: Unit test images match production images exactly
3. **Approach**: devenv.nix declares image version → env vars → testcontainers + docker-compose
4. **Future**: nix generates docker-compose.yaml from same source
5. **Integrity**: sha256 hash verification ensures images are identical

This ensures development, testing, and production environments use identical container images.

## Status

Deferred. Waiting for nix-based docker-compose generation to be implemented first.
